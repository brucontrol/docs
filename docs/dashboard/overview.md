# Dashboard Overview

The **Dashboard** is the main canvas where workspace elements appear as widgets. Each workspace has its own dashboard.

## What You See

- **Widgets** — Each element (button, timer, chart, device control, etc.) renders as a widget
- **Tabs** — When you have multiple workspaces, tabs appear above the dashboard; click to switch
- **Canvas** — Drag, resize, and arrange widgets; zoom and pan for large layouts

## Modes

- **Select mode** — Click widgets to select; use Edit to open the element's properties. On touch devices, drag on empty canvas to pan.
- **Hand mode** — Pan the canvas by dragging (no widget selection). Hand is the default on first load.

Elements are added via the Solution Explorer context menu, not from the dashboard canvas.

## Element Interaction

Widgets show real-time values. Elements with **User Control** enabled let you interact (toggle switches, buttons, sliders) directly from the dashboard. Processes run scripts and automation in the background.
