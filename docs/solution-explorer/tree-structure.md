# Tree Structure

The Solution Explorer tree is organized as follows:

## Root Level

```
├── Workspaces         (dashboards)
│   ├── [workspace folders]
│   └── [workspaces with elements]
├── Processes          (scripts you can run)
│   ├── [script folders]
│   └── [scripts]
├── Data Views         (saved chart configurations)
│   └── [data views]
├── Interfaces         (hardware devices)
│   └── [devices with ports]
├── Mocks              (mock device simulators, when enabled)
│   └── [mock devices]
└── Settings           (application configuration)
    ├── General
    ├── Security
    ├── Services
    ├── License
    ├── Data Storage
    ├── Element Templates
    ├── Device Types
    ├── Plugin Store
    └── System Logs
```

## Workspace Contents

Under each workspace you find:

- **Folders** — Organize elements
- **Elements** — Global Variable, Toggle Switch, Button, Timer, Alarm, Chart, Profile, Script, Generic, Device elements

## Processes Contents

Under Processes you find:

- **Folders** — Organize scripts
- **Scripts** — Automation processes (run, edit, delete)
